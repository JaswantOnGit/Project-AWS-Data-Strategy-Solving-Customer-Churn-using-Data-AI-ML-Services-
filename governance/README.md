# Governance

This folder is where the delivery governance tracker for the AWS customer churn engagement lives.

## What belongs here

- `churn-project-governance-tracker.xlsx` — the master tracker workbook. Add it directly to this folder once it is ready to share, and update this README with a short note on its version/date.

- ## Tracker structure

- The governance tracker workbook is expected to contain the following tabs:

- - **Phase Summary** — one row per delivery phase, with status (Not Started / In Progress / Complete / Blocked), owner, target date, and actual date.
  - - **RAID Log** — Risks, Assumptions, Issues, and Dependencies raised during delivery, each with an owner, severity, and current status.
    - - **Decisions Log** — key decisions made during the engagement, who made them, and the date.
      - - **Action Items** — open action items, owners, and due dates.
        - - **Milestones** — key delivery milestones mapped against the phases described in the main README.
         
          - ## How to use it
         
          - 1. Update the tracker at each governance checkpoint (end of phase, or per the agreed cadence).
            2. 2. Keep the RAID log current — new risks/issues should be added as soon as they're identified, not just before a checkpoint.
               3. 3. Link back to this repository from the tracker so reviewers can find supporting evidence (diagrams, screenshots) easily.
